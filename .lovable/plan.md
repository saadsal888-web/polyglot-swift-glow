
# صفحة عرض القصة

## الفكرة
إنشاء صفحة كاملة لعرض القصة عند الضغط عليها من الصفحة الرئيسية، تعرض فقرات القصة مع الكلمات والأسئلة.

## البيانات المتوفرة في قاعدة البيانات
- **learning_stories**: عنوان القصة (عربي/إنجليزي)، المستوى، المدة، رابط الصوت
- **story_paragraphs**: فقرات القصة بالإنجليزي مرتبة
- **story_vocabulary**: كلمات مهمة مع المعنى بالعربي والنطق
- **story_quiz_questions**: أسئلة اختيار متعدد مع الإجابة الصحيحة والشرح

## تصميم الصفحة (3 مراحل)

### المرحلة 1 - القراءة
- هيدر بعنوان القصة وزر رجوع
- زر تشغيل الصوت (audio player) لسماع القصة
- عرض فقرات القصة بخط واضح مع تأثير ظهور تدريجي
- المستوى والمدة تحت العنوان

### المرحلة 2 - الكلمات المهمة
- قسم "كلمات مهمة" تحت القصة
- كل كلمة تعرض: الكلمة بالإنجليزي، المعنى بالعربي، والنطق

### المرحلة 3 - اختبر فهمك
- زر "اختبر فهمك" ينقل لأسئلة الاختيار المتعدد
- كل سؤال يعرض الخيارات، عند الإجابة يظهر إذا صح أو خطأ مع الشرح
- في النهاية يعرض النتيجة (مثلاً 4/5)

## التغييرات المطلوبة

### 1. ملف جديد: `src/pages/StoryReader.tsx`
- صفحة كاملة تجلب بيانات القصة + الفقرات + الكلمات + الأسئلة من Supabase
- تصميم متوافق مع باقي التطبيق (AppLayout، framer-motion)
- 3 أقسام: القصة، الكلمات، الأسئلة
- زر صوت يستخدم `audio_url` من القصة

### 2. تعديل `src/App.tsx`
- إضافة route جديد: `/stories/:storyId`
- يشير إلى صفحة `StoryReader`
- محمي بـ `ProtectedRoute`
